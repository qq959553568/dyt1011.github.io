邓永涛的博客

一个分享实用经验的技术博客

第一次使用GitHub、Hexo(Node.js),把自己的学习心得写下来，希望能帮助别人，也给自己留个纪念。

采用Hexo+GitHub搭建博客的想法，是我一个朋友推荐我的，所以我就百度了一些教程，再加上向朋友的讨教，所以用Hexo+GitHub的方案搭建了现在大家所阅读到的这个博客，Hexo+GitHub的方案都是开源的，适合我们初学者使用。

这里面的操作基本参照网上的教程，但是单独的某篇讲的又有些问题，所以在这里综合我所遇到的问题，给大家分享一下，如何使用Hexo+GitHub搭建自己的博客

直奔主题，首先安装Git

首先打开Git官网，下载并安装Git，推荐采用默认路径。（Git的功能很多，有兴趣的朋友可以自学)

接下来是Node.js的安装（帮助我们下载Hexo的工具)

Node.js有什么用呢？在本篇文章，它主要是负责帮助我们下载我们所需的Hexo。

第一步：下载并安装Node.js

进入官网，选择自己对应的操作系统，下载好以后自己选个位置（建议提前建立好文件方便储存，这是个好习惯）安装即可。

安装完之后同时按住Win+R，然后输入cmd打开控制台并输入以下代码：

node -v

npm -v

注意node、npm的后面都有一个空格，如果正常安装，输入这两个命令，会显示它们的版本号。

第二步：安装Hexo

我们之前已经安装了Git，所以我们可以使用它的Git Bash Here（右键单击可见），来帮助我们更好的安装Hexo。

首先我们要新建一个存放Hexo文件的目录，自己创建一个自己能找得到的地方，然后在该目录下，右键然后单击Git Bash Here，就可以打开类似控制台的界面，我们就可以输入命令，进行安装，我们输入以下代码：

$npm install hexo-cli -g

$hexo init

$npm install

$hexo g

$hexo s

注意：$是一开始就有的，不用输入，同时注意空格的输入，下文中的代码我将不写$

上述命令输入完毕后，在我们的浏览器地址栏处输入localhost:4000(4000是默认的端口号)，回车之后，就是见证奇迹的时刻，你的第一个博客的模型就出来了。当然现在它只能本地浏览，我们要做的就是让他可以在互联网上被访问。

因此，我们要通过GitHub部署到互联网上

首先，我们要去注册一个GitHub的账号，邮箱尽量填写自己常用的。（在这里推荐大家使用谷歌浏览器，因为它可以自动翻译英语成中文，很方便）

在翻译成中文界面，可以看见下图

单击启动项目，在存储库名称填写：你喜欢的名字.github.io

接下来时创建SHH

在Git Bash（可以直接在桌面使用)输入命令：

$ ssh-keygen -t rsa -C "你的邮箱地址"（注意，C和你的邮箱地址都是要输入的)

接下来的事情，就是连续三次回车（最安全的密码就是没有密码），可以看见在你所Git Bash Here的路径新生成了两个文件，id_rsa.pub，我们用记事本打开，复制里面的全部内容。

紧接着，我们打开GitHub，右上角点击自己的头像，选择设置，然后选择SSH和GPG密钥，

标题可以随便写，在键里面，粘贴我们刚刚复制的内容。然后点击添加SHH密钥

到了发布这个激动人心的时候了

打开我们之前创建的项目，找到_config.yml文件，用记事本打开，在文本的最后我们可以看到

deploy:

type:

我们要修改成这样：

deploy: git

type: git@github.com:qq959553568/dyt1011.github.io.git(这里要换成你的SSH密钥)

repository: master

(注意：在:后面要空格)

你的SSH密钥在你所创建的储存库里面，如下图

修改完成之后，保存即可。然后在我们Hexo项目的目录下，Git Bash以下命令

hexo d(将本地文件传到GitHub储存库)

如果输入后无反应，或者出现ERROR等报错情况，你可以尝试

hexo clean

hexo g -d

或者是

npm install hexo-deployer-git --save

如果还是有问题，删除所建的文件，重新安装Hexo（删除大法）

这样就大功告成了。通过地址

https://自己的用户名/储存库名.github.io/

来访问你自己的博客

当然如果你有自己域名，可以绑定自己的域名。比如，我在腾讯云购买的域名

点击解析之后

你可以模仿我这个填，ip地址可以通过cmd控制台，ping你的博客地址，获取对应的Ip地址，命令如下：

ping 你的博客地址

除此之外，你还需要在你的Hexo本地文件里，找到source文件夹，在里面新建一个文件，名字为CNAME，用记事本打开，在里面输入自己所购买的域名即可。

我自己在做我的博客时候，遇到的最多的问题就是hexo d这个命令传到GitHub储存库中的时候，本地文件有样式，而，GitHub里面只剩文字，这个问题的解决方案就是，打开我们的Hexo项目，以记事本打开_config.yml，找到#URL，然后

url: http://你的域名

root: /

(注意:后面有空格)

然后，你会惊奇的发现，你的第一个博客，可以通过你自己的域名访问了。Hexo有很多主题，而且使用方法百度也有很多内容，大家可以自己去选择自己喜欢的主题。
